# LaTeX-шаблон для исследовательской работы НИЯУ МИФИ

## О проекте

Данный проект представляет собой гибкий LaTeX-шаблон, предназначенный для оформления курсовых и дипломных работ (УИР, НИР, ВКР) в НИЯУ МИФИ.

> **Примеры работы шаблона** можно найти в папке `examples/`:
> - `1-task.pdf` — Сформированное задание.
> - `2-rspz.pdf` — Пример РСПЗ (расширенного содержания).
> - `3-pz.pdf` — Пример финальной пояснительной записки.

Главная цель — избавить студентов от проблем с нормоконтролем и оформлением, предоставив готовый инструмент, соответствующий требованиям университета. Шаблон разработан с упором на гибкость: он подходит для любых тем и позволяет глубокую кастомизацию под нужды конкретного студента.

За основу взят [оригинальный thesis-template](https://gitlab.com/skibcsit/thesis-template/-/tree/006cebaab233165578fafead97208f6a685d22b1/).

## Ключевые особенности

- **Единый стиль**: Оформление соответствует стандартам (ГОСТ) для пояснительных записок.
- **Модульность**: Проект разбит на логические части (Задание, РСПЗ, ПЗ), которые собираются независимо.
- **Автоматизация**: 
  - Автоматический подсчет количества источников, приложений и графиков для реферата.
  - Умная сборка содержания (аннотации разделов включаются в РСПЗ, но скрываются в ПЗ).
- **Удобная конфигурация**: Все настройки (ФИО, тема, подписи) вынесены в отдельные файлы в папке `config/`.
- **Работа с подписями**: Поддержка вставки сканов подписей с настройкой смещения, чтобы не подписывать каждый титульник вручную.
- **Вставка внешних титульных листов**: Возможность замены стандартных титульных листов и задания на сканы (PDF) из сторонних документов. Базовая папка для сканов — `config/assets/titles/`.

## Структура проекта

Проект организован так, чтобы отделить пользовательский контент от настроек шаблона:

1.  **Корневые файлы сборки**:
    -   `1-task.tex` — Задание на работу.
    -   `2-rspz.tex` — Расширенное содержание пояснительной записки (РСПЗ).
    -   `3-pz.tex` — Итоговая пояснительная записка (ПЗ).

2.  **`config/`** — Пользовательские настройки (здесь вы меняете данные под себя):
    -   `template-settings.tex` — Основные настройки (пути к ресурсам, параметры отображения).
    -   `titles-and-members.tex` — Данные для титульных листов (ФИО студента и руководителя, тема работы).
    -   `formal-approvals.tex` — Даты и формальные утверждения.
    -   `assets/signatures/` — Сканы подписей для автоматической вставки.

3.  **`content/`** — Основное содержание работы:
    -   `0-assets/` — Ресурсы: изображения (`images/`), исходные коды (`listings/`) и библиографическая база (`references/bibliography.bib`).
    -   `1-main-content/` — Текст разделов (Введение, Главы, Заключение).
    -   `2-appendices/` — Приложения.

4.  **`template/`** — Внутренние файлы шаблона (стили, макросы, пакеты). Редактировать рекомендуется только опытным пользователям LaTeX.

5.  **`out/`** — Папка, куда по умолчанию сохраняются скомпилированные PDF-файлы.

## Быстрый старт и Базовая Сборка

Для работы с проектом вам понадобится установленный дистрибутив LaTeX (например, TeX Live, MacTeX или MiKTeX) и перл-скрипт `latexmk` (обычно идет в комплекте).

### Базовая сборка через терминал

Рекомендуемый способ базовой сборки — использование `latexmk`. Это гарантирует правильное разрешение перекрестных ссылок и библиографии.

Шаблон настроен на сборку в папку `out/`, чтобы временные файлы не засоряли проект.

**Команда для сборки:**

```bash
latexmk -xelatex -outdir=out/ -auxdir=out/.latex_tmp_files/ <имя_файла>.tex
```

**Примеры:**

1.  **Собрать Задание**:
    ```bash
    latexmk -xelatex -outdir=out/ -auxdir=out/.latex_tmp_files/ 1-task.tex
    ```
2.  **Собрать РСПЗ**:
    ```bash
    latexmk -xelatex -outdir=out/ -auxdir=out/.latex_tmp_files/ 2-rspz.tex
    ```
3.  **Собрать ПЗ**:
    ```bash
    latexmk -xelatex -outdir=out/ -auxdir=out/.latex_tmp_files/ 3-pz.tex
    ```

Сгенерированные PDF файлы (`1-task.pdf`, `2-rspz.pdf`, `3-pz.pdf`) появятся в папке `out/`.

### Более продвинутая сборка и более продвинутый старт

Для наиболее комфортной и быстрой работы рекомендуется использовать следующую связку инструментов:

1.  **Редактор: VS Code**
    В качестве редактора `.tex` файлов удобно использовать [VS Code](https://code.visualstudio.com/). Это современный, легкий и расширяемый редактор.

2.  **Плагин: LaTeX Workshop**
    Практически все задачи по сборке и просмотру берет на себя расширение [LaTeX Workshop](https://marketplace.visualstudio.com/items?itemName=James-Yu.latex-workshop). Оно автоматически компилирует документ при сохранении, поддерживает SyncTeX (переход от PDF к коду и обратно) и многое другое.

3.  **Дистрибутив: BasicTeX (для macOS)**
    Вместо полного пакета MacTeX (который весит несколько гигабайт) можно использовать **BasicTeX**.
    -   **Преимущества**: Весит всего 100–200 МБ.
    -   **Недостатки**: "Из коробки" содержит минимум пакетов. При первой сборке шаблона возникнут ошибки из-за нехватки стилей.
    -   **Решение**: Недостающие пакеты устанавливаются через утилиту `tlmgr` (входит в комплект). Как именно установить тот или иной пакет на вашей ОС — лучше всего подскажет ChatGPT или поиск по ошибке компилятора. Он умеет генерировать команды для установки списка пакетов.

4.  **Плагин: LaTeX Utilities**
    Расширение [LaTeX Utilities](https://marketplace.visualstudio.com/items?itemName=tecosaur.latex-utilities) дополняет функционал Workshop. Оно упрощает вставку таблиц, форматирование и связывает редактор с Zotero.

5.  **Работа с литературой: Zotero**
    [Zotero](https://www.zotero.org/) — мощный менеджер библиографии.
    -   Установите **Zotero Connector** в браузер — это позволит добавлять источники (статьи, книги, веб-страницы) в базу одним кликом.
    -   Экспортируйте библиотеку в формат BibTeX и подключите к проекту. Это "оооооочень" сильно упрощает поиск и занесение литературы.

6.  **Использование AI ассистентов**
    Когда весь проект находится в LaTeX, инструменты вроде **GitHub Copilot** или **Cursor** могут значительно ускорить написание текста работы (УИР/НИР/ВКР), предлагая продолжения фраз и целые блоки кода.
    *Примечание*: Итоговые PDF, написанные с помощью AI, не проверялись на антиплагиат в рамках этого шаблона. Используйте с осторожностью.

7.  **Поддержка**
    Если есть вопросы по настройке окружения или шаблона — можно смело обращаться, подскажу по своим конфигурациям.
## Настройка проекта

Все основные настройки находятся в папке `config/`.

### 1. Персональные данные и Тема
Файл `config/titles-and-members.tex`.
Здесь задаются:
-   `\studentName`: ФИО студента.
-   `\supervisorName`: ФИО руководителя.
-   `\workTitle`: Тема работы.
-   Другие данные для титульных листов (группа, кафедра и т.д.).

### 2. Подписи
Файл `config/assets/signatures/` и использование в коде.
-   Положите сканы подписей (PNG с прозрачным фоном) в `config/assets/signatures/`.
-   Для вставки подписи используйте макрос `\signat` (см. примеры в `content/1-main-content/01-task-content.tex`).
-   Вы можете настроить смещение (`xsign`, `ysign`) и масштаб (`scale`) для каждой подписи индивидуально, чтобы она ровно "встала" на строку.

### 3. Даты и Утверждения
Файл `config/formal-approvals.tex`.
Здесь задаются даты выдачи задания, утверждения и защиты.

### 4. Библиография
Файл `content/0-assets/references/bibliography.bib`.
Добавляйте источники в формате BibTeX. Стиль оформления ссылок уже настроен в соответствии с ГОСТ.
